В корне лежит backUp БД, название файла cinema.sql.
Создаём БД `cinema` и импортируем в неё данный backUp.
Заливаем файлы сайта. Данный сайт реализован по методологии MVC (/application/controllers|models|views)
Проверяем настройки коннекта к БД в файле /db/pdoconnect.php

Доступы к админпанели, 2 тестовых пользователя (хранятся в файле /db/user.json):
    login: max, pass: max
    login: root, pass: root

Структура БД, таблицы:
    films - хранится информация о фильмах.
    seances - справочник, возможные варианты сеансов фильмов.
    seances_to_film - таблица зависимости фильмов и сеансов к ним.
    tikets - список купленных билетов с информацией о пользователе/ По этой таблице выясняются занятые места в зале.



P.S.
Т.к. в ТЗ не оговаривалось стоимость билетов, этот функционал не реализовывался, хотя он логичен.
